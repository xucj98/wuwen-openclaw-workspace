## 重要发现

### Compaction 与 Bootstrap 文件

- **Bootstrap 文件在每次 model run 时从磁盘重新加载**，不是只在 session 启动时加载
- 这意味着 compaction 后，bootstrap 文件（SOUL.md, USER.md, MEMORY.md 等）不会丢失
- 对话历史被压缩，但文件内容始终从磁盘读取

### Token 统计

- `openclaw sessions --json` 的 inputTokens/outputTokens 是当前 context 的估算，不是累计
- **真实的 token 消耗在 JSONL 文件里**，每条消息的 usage 字段
- 今天累计：~11.4M input tokens, ~62K output tokens

### Heartbeat vs Cron

- Heartbeat：定期轮询，有对话上下文，适合批量检查
- Cron：精确时间触发，独立 session，适合定时任务

## 学到的教训

- 耗时任务应该派生 subagent，不要让用户等待
- Memory 文件应该分类：USER.md（用户信息）、MEMORY.md（经验教训）、每日日志（活动流水）
- Bootstrap 文件是持久层，对话历史是易失层
