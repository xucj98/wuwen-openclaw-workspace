### 飞书文档写入权限问题已解决

**问题**：`feishu_doc write` 和 `append` 操作返回 400 错误

**根本原因**：缺少 `docx:document.block:convert` 权限

**解决方案**：在飞书管理后台添加权限后，write/append 操作正常工作

**关键发现**：
- SDK 路径 `/open-apis/docx/v1/documents/blocks/convert` 是正确的
- 之前错误分析是 SDK bug，实际是权限问题
- 需要的权限：`docx:document.block:convert`

### Subagent Compaction 不触发问题调研

**问题现象**：
- Subagent 的 `inputTokens` 可达到 1.3M，远超 contextWindow（203k）
- 但 `compactionCount` 始终为 0
- 实际 context 使用量严重超标

**可能原因**：

1. **`contextTokens` 来源问题**：
   - Pi 使用 API 返回的 `usage.totalTokens` 作为 `contextTokens`
   - 但某些 API 代理可能返回的是当前请求的 tokens，不是累计 context 占用量
   - 导致 `contextTokens` 远小于实际 context

2. **Compaction 触发条件**：
   ```
   contextTokens > contextWindow - reserveTokens
   ```
   如果 `contextTokens` 不准确，条件永远不满足

3. **相关 GitHub Issues**：
   - Issue #45268: context usage is misleading when totalTokensFresh=false
   - Issue #45513: Token count mismatch between views

**待确认**：API 代理是否正确返回了 context tokens
