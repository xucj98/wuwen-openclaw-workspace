# VLM机器人数据质量评估调研规范

## 调研目标

系统性地梳理基于 VLM 的机器人数据质量评估方法，重点关注开源工作，为构建 benchmark 提供参考。

---

## 调研表格规范

每篇论文需要填写以下字段：

### 1. 基本信息

| 字段 | 说明 | 示例 |
|------|------|------|
| 论文名称 | 完整标题 | Unpacking Failure Modes of Generative Policies |
| 作者/机构 | 主要作者和所属机构 | UC Berkeley, Stanford |
| 发表时间 | 会议/期刊/年份 | CoRL 2024 |
| 开源状态 | 代码/数据是否开源 | ✅ 代码开源 (GitHub链接) / ❌ 未开源 |
| 论文链接 | PDF 或 arXiv 链接 | https://arxiv.org/abs/xxx |

### 2. 方法概述

| 字段 | 说明 |
|------|------|
| 使用的 VLM | 具体模型名称（如 GPT-4o, Claude 3.5, Qwen2-VL）|
| 任务定义 | VLM 被用来做什么（失败检测、质量评分、数据筛选等）|
| 输入格式 | VLM 接收什么输入（关键帧、视频、轨迹、状态序列等）|
| 输出格式 | VLM 输出什么（分类标签、评分、文本描述等）|

### 3. Prompt 设计（核心）

| 字段 | 说明 |
|------|------|
| Prompt 完整内容 | 原文中的 prompt 模板（如有）|
| Prompt 设计思路 | 为什么这样设计 prompt |
| Prompt 变体 | 是否尝试了不同的 prompt 设计 |
| Prompt 长度 | 大约的 token 数量 |

**如果论文没有详细说明 prompt 设计，标注"未详细说明"并尝试从代码中提取。**

### 4. Benchmark 设计（核心 - 需要详尽说明）

#### 4.1 数据来源

| 字段 | 说明 |
|------|------|
| 数据集名称 | 使用的数据集名称 |
| 数据来源 | 自己采集 / 开源数据集 / 混合 |
| 场景类型 | 仿真环境 / 真实机器人 / 混合 |
| 任务类型 | 具体任务列表 |
| 数据量 | 轨迹数量、帧数、时长等 |

#### 4.2 数据标注

| 字段 | 说明 |
|------|------|
| 标注类型 | 成功/失败标签 / 质量评分 / 失败原因 / 多标签 |
| 标注方式 | 人工标注 / 自动生成 / 规则生成 |
| 标注标准 | 如何定义"成功"/"失败"/"次优" |
| 标注者数量 | 多少人参与标注 |
| 标注一致性 | inter-annotator agreement（如有）|

#### 4.3 数据示例

**必须提供至少 1-2 个具体的数据样例**：

```
样例 1：
- 任务：把箱子盖上
- 输入：5 帧关键帧图像 + 文本描述
- 真实标签：失败（盖子卡住）
- VLM 预测：失败
- VLM 解释：盖子未能完全闭合，存在障碍物

样例 2：
...
```

#### 4.4 评价指标

| 字段 | 说明 |
|------|------|
| 指标名称 | 具体指标（Accuracy, F1, AUC 等）|
| 指标定义 | 数学定义或计算方式 |
| 指标选择理由 | 为什么选择这个指标 |
| 基线对比 | 与哪些方法对比 |

### 5. 实验结果

| 字段 | 说明 |
|------|------|
| 主要结果 | 核心指标数值 |
| 消融实验 | 是否有消融实验，结论是什么 |
| 失败案例分析 | 哪些情况下效果不好 |
| 泛化性测试 | 是否测试了跨任务/跨场景泛化 |

### 6. 其他注意点

| 字段 | 说明 |
|------|------|
| 关键 Trick | 论文中提到的关键技巧 |
| 局限性 | 论文自己指出的或你发现的局限 |
| 复现难度 | 估计的复现难度（高/中/低）|
| 相关工作 | 可以一起看的其他论文 |

---

## 调研产出

1. **调研表格**：按上述规范填写的表格（飞书文档）
2. **方法分类**：将调研的方法按技术路线分类
3. **Gap 分析**：现有方法的不足和潜在改进方向
4. **Benchmark 设计建议**：基于调研，提出 benchmark 设计建议

---

## 调研范围

### 必须调研

- 明确使用 VLM 进行机器人数据质量评估的论文
- 开源代码或数据的工作
- 近 2 年的工作（2024-2026）

### 可以扩展

- VLM 用于机器人失败检测
- VLM 用于数据标注/筛选
- VLA 作为数据裁判（如 VLA-as-a-Judge）
- VideoQA 用于机器人任务评估

### 暂不调研

- 纯 LLM 方法（不涉及视觉）
- 非数据质量评估的 VLM 应用

---

## 时间规划

- 第 1 周：完成文献检索和初步筛选
- 第 2 周：按规范填写调研表格
- 第 3 周：方法分类和 Gap 分析
- 第 4 周：Benchmark 设计建议
